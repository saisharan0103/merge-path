Conflicts between spec and existing code

1. Database: SQLite (current) vs Postgres (spec).
Spec says Postgres 15+. Current code uses SQLite (sqlite:///./data/patchpilot.db) and inits via init_db() at startup. Alembic isn't set up. Before phase-1 changes:

2. Async vs sync SQLAlchemy.
Spec says "async everywhere on backend." Current code is fully sync (Session = Depends(get_db)). Switching mid-build is annoying. Pick one:

Migrate to async now (one-time pain, matches spec)
Stay sync, update spec I'd recommend migrate to async now, before more sync-style services are written.

Current schema has RepositoryConfig (per-repo allowed_labels, max_files_changed, auto_pr_enabled, etc.).
